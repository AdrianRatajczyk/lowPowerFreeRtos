01161 /*
01162  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE.  IT IS ONLY

01166  * Called from the real time kernel tick (either preemptive or cooperative),

01173 /*
01174  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE.  IT IS AN
01175  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
01176  *
01177  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
01178  *
01179  * Removes the calling task from the ready list and places it both
01180  * on the list of tasks waiting for a particular event, and the
01181  * list of delayed tasks.  The task will be removed from both lists
01182  * and replaced on the ready list should either the event occur (and
01183  * there be no higher priority tasks waiting on the same event) or
01184  * the delay period expires.
01185  *
01186  * @param pxEventList The list containing tasks that are blocked waiting
01187  * for the event to occur.
01188  *
01189  * @param xTicksToWait The maximum amount of time that the task should wait
01190  * for the event to occur.  This is specified in kernel ticks,the constant
01191  * portTICK_RATE_MS can be used to convert kernel ticks into a real time
01192  * period.
01193  */
01194 void vTaskPlaceOnEventList( const xList * const pxEventList, portTickType xTicksToWait ) PRIVILEGED_FUNCTION;
01195 
01196 /*
01197  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE.  IT IS AN
01198  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
01199  *
01200  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
01201  *
01202  * This function performs nearly the same function as vTaskPlaceOnEventList().
01203  * The difference being that this function does not permit tasks to block
